# TODO Manager Skill

> 管理项目 TODO.md，追踪任务完成状态

## 触发时机

当用户提及以下内容时触发：
- "TODO"、"待办"、"任务列表"
- "完成了 xxx"、"做完了 xxx"
- "更新进度"、"标记完成"
- "本周任务"、"当前任务"

## 功能

### 1. 加载 TODO

读取 `/TODO.md` 文件并解析：
- 高优先级任务
- 中优先级任务
- 低优先级任务
- 进度追踪表格

### 2. 更新任务状态

将任务从 `- [ ]` 更新为 `- [x]`：

**更新规则：**
1. 精确匹配任务描述
2. 更新父任务（如果所有子任务完成）
3. 更新进度追踪表格中的百分比

### 3. 更新进度表格

自动计算并更新：
```markdown
| 模块 | 进度 | 状态 | 说明 |
|------|------|------|------|
| vvce-core | XX% | ✅/🔄/🔴 | ... |
```

**状态图标：**
- `✅` 已完成 (100%)
- `🔄` 进行中 (1-99%)
- `🔴` 未开始 (0%)
- `⏳` 待定

### 4. 更新日期

每次修改后更新：
```markdown
> **最后更新**: YYYY-MM-DD
```

## 工作流程

### 查看 TODO

```
用户: 看看 TODO
AI: [读取 TODO.md，展示摘要]

=== 当前 TODO 状态 ===

📊 M0 阶段完成度：70%

🔥 高优先级任务：
  vvce-schema: 85% ✅
  vvce-components: 5% 🔴 ← 关键路径
  Backend API: 30% 🔄

📋 本周重点：
  1. [ ] 组件协议定义
  2. [ ] 组件注册表
  3. [ ] Dialog 组件
```

### 标记完成

```
用户: 我完成了 Dialog 组件的基础渲染
AI: [更新 TODO.md]

✅ 已标记完成：
  - [x] 支持 `text` 属性渲染

📊 更新进度：
  - Dialog 组件: 10% → 20%
  - vvce-components: 5% → 8%
```

### 添加新任务

```
用户: 添加一个任务：修复 Store 的性能问题
AI: [在合适的分类下添加任务]

✅ 已添加任务到 "技术债务" 分类：
  - [ ] 修复：Store 的性能问题
```

## TODO.md 结构

```markdown
# VV Education - TODO 清单

> **最后更新**: YYYY-MM-DD
> **当前阶段**: M0（核心引擎开发）

## 🔥 高优先级（M0 完成前）
### 模块名称
- [ ] **任务标题** `@priority:high`
  - [ ] 子任务 1
  - [ ] 子任务 2

## 📋 中优先级（M0 完成后）
...

## 🌟 低优先级（M1/M2 阶段）
...

## 🐛 已知问题 & 技术债务
...

## 📊 进度追踪
| 模块 | 进度 | 状态 | 说明 |
|------|------|------|------|
...
```

## 进度计算规则

1. **子任务完成率** = 已完成子任务数 / 总子任务数
2. **模块完成率** = 所有任务完成率的加权平均
3. **阶段完成率** = 所有模块完成率的加权平均

权重按优先级：
- `@priority:critical`: 3x
- `@priority:high`: 2x
- `@priority:medium`: 1x
- `@priority:low`: 0.5x

## 注意事项

- 保持 TODO.md 格式一致
- 每次更新后提交到 git
- 不要删除已完成的任务，保留作为记录
- 使用相对路径引用代码文件
